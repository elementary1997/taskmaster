## Введение

Я долго использовал Google Tasks для управления своими делами. Удобный интерфейс, синхронизация между устройствами, интеграция с Gmail и Google Calendar — все было хорошо. Но была одна проблема: **у Google Tasks нет десктопного приложения**.

Приходилось либо открывать браузер и держать вкладку с Google Tasks, либо использовать веб-версию в отдельном окне. Это неудобно: браузер занимает место, постоянно нужно переключаться между вкладками, а веб-версия не всегда работает офлайн.

Я решил сделать свой менеджер задач для десктопа. Не копию Google Tasks, а что-то свое — простое, красивое и удобное. В этой статье расскажу, как это получилось и что использовал для разработки.

## Что я хотел получить

Мне нужен был менеджер задач, который:

- **Работает как нативное приложение** — не браузер, а настоящая программа для Windows/Linux/macOS
- **Не мешает работать** — полупрозрачное окно, можно поставить в угол экрана и забыть
- **Быстро запускается** — не нужно ждать загрузки браузера, работает сразу
- **Работает без интернета** — все данные хранятся на компьютере в обычных файлах, не нужен постоянный доступ в сеть
- **Красиво выглядит** — современный дизайн с эффектом "стекла" и разными темами
- **Простой в использовании** — понятный интерфейс, ничего лишнего, как в Google Tasks, но лучше

Главное отличие от Google Tasks — это полноценное десктопное приложение, которое не зависит от браузера и интернета. Плюс добавил функции, которых нет в Google Tasks: таймеры Pomodoro, теги, отчеты о времени, темы оформления.

## Как я это делал

### Что использовал

Я выбрал **PySide6** (это Qt для Python) потому что:

1. **Работает везде** — на Windows, Linux и macOS
2. **Красивые окна** — можно делать прозрачность, тени, анимации
3. **Быстро работает** — использует нативные возможности системы
4. **Гибкий** — можно делать свои элементы интерфейса

### Разработка в Cursor

Весь проект я разрабатывал в **Cursor** — это современный редактор кода с встроенным ИИ-ассистентом. Это очень помогло:

- **Быстрое написание кода** — ИИ предлагал готовые решения для типовых задач
- **Исправление ошибок** — когда что-то не работало, можно было просто спросить у ассистента
- **Рефакторинг** — легко переписывать и улучшать код
- **Документация** — ассистент помогал разобраться с PySide6, когда не знал, как что-то сделать

Особенно полезно было при работе с Qt — там много классов и методов, и не всегда понятно, что использовать. Cursor помогал находить нужные решения быстро.

### Как хранятся данные

Каждая задача — это просто структура с полями:

```python
@dataclass
class Task:
    id: int              # Уникальный номер
    title: str           # Название задачи
    description: str     # Описание (может быть пустым)
    priority: str        # Приоритет: high, medium, low
    status: str         # Выполнено или нет
    due_date: str       # Срок выполнения
    created: str        # Когда создана
    tags: List[str]     # Теги для организации
```

Все задачи сохраняются в обычный JSON-файл. Это удобно:
- Можно открыть в любом текстовом редакторе
- Легко скопировать на другой компьютер
- Можно редактировать вручную, если нужно

## Как это работает внутри

### Масштабирование интерфейса

Чтобы приложение хорошо выглядело на любом экране (большом или маленьком), я сделал систему масштабирования:

```python
class ZoomManager:
    _scale = 1.0  # Коэффициент масштаба
    
    @classmethod
    def scaled(cls, value):
        return int(value * cls._scale)  # Умножаем размер на коэффициент
```

Все размеры (кнопки, отступы, шрифты) умножаются на этот коэффициент. Если экран большой — можно увеличить масштаб, если маленький — уменьшить. Все элементы остаются пропорциональными.

### Как работают темы

Каждая тема — это просто словарь с цветами:

```python
AVAILABLE_THEMES = {
    "Сапфировая ночь": {
        'window_bg_start': "#1a1a2e",  # Цвет фона сверху
        'window_bg_end': "#16213e",     # Цвет фона снизу
        'text_primary': "#ffffff",      # Цвет текста
        # ... и так далее для всех элементов
    }
}
```

Когда пользователь выбирает тему, все элементы интерфейса перекрашиваются в новые цвета. Qt сам обновляет все виджеты.

### Где хранятся данные

Все задачи сохраняются в специальной папке пользователя:
- **Windows**: `C:\Users\ВашеИмя\AppData\Local\ModernTaskManager\`
- **Linux**: `~/.local/share/ModernTaskManager/`

Это удобно:
- Данные не потеряются при обновлении программы
- Можно легко скопировать папку на другой компьютер
- Файлы не мешаются в рабочей директории

### Как работает перетаскивание

Когда пользователь нажимает на задачу и начинает тащить, Qt создает объект "перетаскивания":

```python
def mousePressEvent(self, event):
    if event.button() == Qt.LeftButton:
        drag = QDrag(self)  # Создаем объект перетаскивания
        mime_data = QMimeData()
        mime_data.setText(str(self.task.id))  # Сохраняем ID задачи
        drag.setMimeData(mime_data)
        drag.exec()  # Запускаем перетаскивание
```

В Qt это встроено, нужно только правильно обработать события. Когда задача "упадет" на нужное место, приложение получает ID и перемещает задачу.

## Оптимизация

### Размер программы

Для Windows программа собирается в один файл `.exe` размером около 60 МБ. Это нормально для приложения на Python с Qt. Чтобы уменьшить размер:
- Использую PyInstaller с оптимизацией
- Исключаю ненужные модули Qt
- Сжимаю ресурсы (иконки, звуки)

### Скорость запуска

Программа запускается меньше чем за 2 секунды:
- Данные загружаются только когда нужно
- Интерфейс инициализируется быстро
- Минимум зависимостей

### Память

Программа не жрет много памяти:
- В интерфейсе показываются только активные задачи
- Если задач много, используется прокрутка
- Данные хранятся компактно в JSON

## Заключение

Этот проект родился из простой потребности — иметь нормальное десктопное приложение для управления задачами. Google Tasks хорош, но без десктопной версии он не всегда удобен для работы за компьютером.

Я сделал свое решение, которое:
- Работает как нативное приложение (не браузер)
